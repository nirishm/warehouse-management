Buttons

Section 04 — 7 variants · 3 sizes · 4 states

CTA Rule
Primary CTA buttons are always pill-shaped (border-radius: 9999px) with active:scale(0.98). This is the single most distinctive button in the system.
Variants — All at default (lg = 48px) size

Design Principles

Section 12 — 7 rules for LLMs implementing WareOS UI

  1. White cards on off-white pages. Page background is always --bg-off #F5F5F3. Cards and panels are always --bg-base #FFFFFF. Never invert this.
  2. Typography roles are absolute. Hedvig Letters Serif = page titles + KPI display values only. Space Mono = table headers (10px, uppercase, 0.08em tracking, --text-dim), sequence numbers (bold 11–12px, accent color on the active entry), timestamps, badges. Rethink Sans = all other UI text.
  3. Orange marks every interaction point. CTA buttons are pills: height: 48px; border-radius: 9999px; active:scale(0.98). Focused inputs: border-color: var(--accent); box-shadow: 0 0 0 3px rgba(244,95,0,0.1). These are non-negotiable.
  4. Orange left-border = active nav. The sidebar active state is: border-left: 2px solid var(--accent) + background: var(--accent-tint) + color: var(--accent). All three together, never partially.
  5. Status badges use pill shape; type labels use rect. .badge (status: received/dispatched/confirmed/cancelled) = border-radius: 9999px. .type-badge (DISPATCH/PURCHASE/SALE) = border-radius: 4px. Do not mix.
  6. Always use --accent for the brand orange (#F45F00). Use --accent-tint (rgba(244,95,0,0.06)) for the 6% tint surface. These are the two canonical orange tokens in the design system.
  7. Never use inline color values; always use tokens. Every color reference must go through a CSS custom property. This ensures theme consistency and makes the system legible to automated tools. Exception: brand-hex values in comments for documentation.
